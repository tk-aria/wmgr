1//! # wmgr - Git Repository Manager
2//!
3//! `wmgr` is a command-line tool for managing multiple git repositories organized in workspaces.
4//! It provides a clean, efficient way to synchronize, check status, and run commands across
5//! collections of repositories.
6//!
7//! ## Features
8//!
9//! - **Repository Management**: Clone, sync, and manage multiple git repositories
10//! - **Group Organization**: Organize repositories into logical groups
11//! - **Parallel Operations**: Execute commands across repositories in parallel
12//! - **Status Checking**: Get an overview of all repository states
13//! - **Manifest-based Configuration**: Define your workspace using YAML manifests
14//!
15//! ## Quick Start
16//!
17//! 1. Create a manifest file (`manifest.yml`):
18//!
19//! ```yaml
20//! repos:
21//!   - dest: "frontend"
22//!     url: "https://github.com/example/frontend.git"
23//!     groups: ["web"]
24//!   - dest: "backend"
25//!     url: "https://github.com/example/backend.git"
26//!     groups: ["api"]
27//! ```
28//!
29//! 2. Initialize your workspace:
30//!
31//! ```bash
32//! wmgr init manifest.yml
33//! ```
34//!
35//! 3. Sync all repositories:
36//!
37//! ```bash
38//! wmgr sync
39//! ```
40//!
41//! ## Architecture
42//!
43//! The crate is organized using clean architecture principles:
44//!
45//! - [`domain`]: Core business logic and entities
46//! - [`application`]: Use cases and business workflows  
47//! - [`infrastructure`]: External dependencies and I/O operations
48//! - [`presentation`]: CLI interface and user interaction
49//! - [`common`]: Shared utilities and error handling
50//!
51//! ## Domain Model
52//!
53//! The core domain entities include:
54//!
55//! - [`domain::entities::workspace::Workspace`]: Represents a workspace containing multiple repositories
56//! - [`domain::entities::manifest::Manifest`]: Configuration defining repositories and their organization
57//! - [`domain::entities::repository::Repository`]: Individual git repository with its configuration
58//! - [`domain::value_objects::git_url::GitUrl`]: Type-safe Git URL representation
59//! - [`domain::value_objects::branch_name::BranchName`]: Type-safe branch name representation
60//!
61//! ## Use Cases
62//!
63//! The main application use cases are:
64//!
65//! - [`application::use_cases::init_workspace`]: Initialize a new workspace from a manifest
66//! - [`application::use_cases::sync_repositories`]: Synchronize repositories in a workspace
67//! - [`application::use_cases::status_check`]: Check the status of repositories
68//! - [`application::use_cases::foreach_command`]: Execute commands across repositories
69//!
70//! ## Infrastructure
71//!
72//! Infrastructure components provide concrete implementations:
73//!
74//! - [`infrastructure::git`]: Git operations using libgit2
75//! - [`infrastructure::filesystem`]: File system operations and configuration storage
76//! - [`infrastructure::process`]: Process execution for external commands
77//!
78//! ## Error Handling
79//!
80//! The crate uses a comprehensive error handling system:
81//!
82//! - [`common::error::TsrcError`]: Main error type with detailed context
83//! - [`common::result::TsrcResult`]: Type alias for `Result<T, TsrcError>`
84//!
85//! ## Examples
86//!
87//! ### Using the Library
88//!
89//! ```rust,no_run
90//! use wmgr::application::use_cases::sync_repositories::{
91//!     SyncRepositoriesUseCase, SyncRepositoriesConfig
92//! };
93//! use wmgr::domain::entities::workspace::Workspace;
94//!
95//! # async fn example() -> wmgr::Result<()> {
96//! // Load workspace
97//! let workspace = Workspace::load_from_path(".")?;
98//!
99//! // Create sync configuration
100//! let config = SyncRepositoriesConfig::new(".")
101//!     .with_groups(vec!["web".to_string()])
102//!     .with_force(false);
103//!
104//! // Execute sync
105//! let use_case = SyncRepositoriesUseCase::new(config);
106//! let result = use_case.execute().await?;
107//!
108//! println!("Synced {} repositories", result.successful);
109//! # Ok(())
110//! # }
111//! ```
112//!
113//! ### Working with Manifests
114//!
115//! ```rust,no_run
116//! use wmgr::application::services::manifest_service::ManifestService;
117//! use std::path::Path;
118//!
119//! # async fn example() -> wmgr::Result<()> {
120//! let manifest_service = ManifestService::new();
121//! let processed = manifest_service.parse_from_file(Path::new("manifest.yml")).await?;
122//!
123//! println!("Found {} repositories", processed.manifest.repos.len());
124//!
125//! for repo in &processed.manifest.repos {
126//!     println!("Repository: {} -> {}", repo.dest, repo.url);
127//! }
128//! # Ok(())
129//! # }
130//! ```
131//!

135//! for different backends or additional functionality.
